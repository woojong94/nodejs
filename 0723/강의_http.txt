http 모듈 - 웹서버 구축을 위한 모듈

1. 요청(request), 응답(response)
	Request		- 사용자의 요청
	Response	- 사용자의 응답
	
2. 헤더	- 요청, 응답에 대한 정보가 담겨있는 부분

	Request		- 사용자가 요청한 주소
					- 사용자가 요청한 데이터(GET, POST 등의 데이터, COOKIE)
					- 환경(브라우저의 종류, IP ...)
					- 전송방식(method-GET, POST)
	
	Response 	- 서버가 출력하는 페이지에 대한 정보
	
	
	참고) 
	-> HTML이 출력 되기 전에 Response 헤더가 먼저 출력
	-> HTML이 출력 된 이후에 Response 헤더는 출력될 수 없다
	
	출력이 완료되면 하단 부분 출력 방지를 위해
	return을 하여 return 이후가 출력되는 것을 방지
	
3. 응답 코드
	요청 --> 응답
	
	20X	--> 정상 응답
	
	30X	--> 페이지 이동관련 오류(URL)
				-> 304 오류 -> cache
					-> 브라우저는 한번 다운로드 받은 웹 자원(.js, .css, img, video ...)을 일정 시간동안 보관
						-> 동일한 요청 파일이 있는 경우 저장된 파일로 로딩
						-> 성능이 개선
	
	40X	--> 요청 쪽(Request)에 이상이 있다(사용자 쪽에 문제가 있는 경우)
	
	50X	--> 응답하는 쪽에 이상이 있다(서버에 문제가 있는 경우)
